## External Services & Infrastructure

Mocha uses several external services for continuous integration, code quality, and project management:

### Codecov

**Purpose**: Code coverage tracking and reporting

- **Dashboard**: [app.codecov.io/gh/mochajs/mocha](https://app.codecov.io/gh/mochajs/mocha)
- **Integration**: Automatic coverage reports on all PRs
- **Requirement**: PRs must not decrease overall code coverage
- **Setup**: Contributors should [add the Codecov GitHub app](https://app.codecov.io/login) to upload coverage from their forks (recommended but not required to open a PR)

### EasyCLA

**Purpose**: Contributor License Agreement (CLA) management

- **CLA Portal**: [cla.js.foundation/mochajs/mocha](https://cla.js.foundation/mochajs/mocha)
- **Requirement**: All code contributions require a signed CLA
- **Process**: A bot will prompt you to sign when you open your first PR
- **Documentation**: [Linux Foundation EasyCLA Docs](https://docs.linuxfoundation.org/lfx/easycla/v2-current/contributors)

### GitHub Actions

**Purpose**: Continuous Integration and automated workflows

- **CI Pipeline**: All tests, linting, and checks run automatically on PRs
- **Workflows**: Located in `.github/workflows/`
- **View Results**: [github.com/mochajs/mocha/actions](https://github.com/mochajs/mocha/actions)

#### Key Workflows

- `mocha.yml`: Main test suite (Node.js & browser tests, linting, formatting)
- `npm-script.yml`: Reusable workflow runner with Codecov integration
- `release-please.yml`: Automated release management
- `nightly-site-deploy.yml`: Automated documentation site updates

### Netlify

**Purpose**: Documentation website hosting and deployment

- **Live Site**: [mochajs.org](https://mochajs.org)
- **Configuration**: `docs-next/netlify.toml`
- **Deploy Previews**: Automatic preview deployments for documentation PRs
- **Nightly Builds**: Automated daily deployments to update supporter information

## Repository Configuration Files

Understanding the various configuration files in the repository:

### Test & Build Configuration

- **`.mocharc.yml`**: Mocha's own test configuration
- **`.nycrc`**: Code coverage configuration (NYC/Istanbul)
- **`karma.conf.js`**: Browser testing configuration
- **`rollup.config.mjs`**: Build configuration for browser bundle
- **`tsconfig.json`**: TypeScript configuration for type checking

### Code Quality & Formatting

- **`eslint.config.js`**: ESLint linting rules
- **`.prettierignore`**: Files excluded from Prettier formatting
- **`.editorconfig`**: Editor configuration for consistent code style
- **`.lintstagedrc.json`**: Pre-commit hook configuration for staged files

### Dependency Management

- **`.fossaignore`**: FOSSA license scanning exclusions (excludes docs and example files from license compliance checks)
- **`.knip.jsonc`**: Dead code detection configuration
- **`.browserslistrc`**: Browser compatibility targets

### Development Environment

- **`.wallaby.js`**: Wallaby.js test runner configuration (for IDE integration)
- **`.npmrc`**: npm configuration

## Repository Tooling

Mocha uses various tools to maintain code quality and consistency:

### Linting & Formatting

- **ESLint**: JavaScript linting with modern flat config format
- **Prettier**: Code formatting (runs automatically on commit via lint-staged)
- **lint-staged**: Pre-commit hooks to format staged files

### Testing

- **Mocha**: Yes, Mocha tests itself! (See `.mocharc.yml`)
- **NYC**: Code coverage measurement
- **Karma**: Browser test runner
- **Playwright**: Browser automation for integration tests

### Build & Package

- **Rollup**: Bundles the browser version of Mocha
- **TypeScript**: Type checking (no transpilation, Node.js native support)

### Documentation

- **Astro + Starlight**: Static site generator for documentation (in `docs-next/`)
- **MDX**: Markdown with JSX for documentation content

### Dependency Analysis

- **FOSSA**: License compliance scanning
- **Knip**: Dead code and unused dependency detection

## Useful npm Scripts

Key scripts that contributors should know about:

### Testing

```bash
npm test                 # Run all tests
npm run test-node        # Run Node.js tests only
npm run test-browser     # Run browser tests only
npm run test-smoke       # Run smoke tests
npm run test:parallel    # Run parallel mode tests
```

### Code Quality

```bash
npm run lint             # Run ESLint
npm run format:check     # Check Prettier formatting
npm run format:fix       # Fix Prettier formatting
```

### Build

```bash
npm run build            # Build browser bundle (mocha.js)
npm run clean            # Remove built files
```

### Documentation

```bash
npm run docs             # Build documentation site
npm run docs:preview     # Preview documentation locally
```

### Development

```bash
npm run test:watch       # Run tests in watch mode
npm run prebuildTypes    # Generate TypeScript type definitions
```

## Project Structure

- **`test/`**: Test suite
- **`docs-next/`**: Documentation site source (Astro + Starlight)

## Development Workflow

1. **Fork and clone** the repository
2. **Install dependencies**: `npm install`
3. **Create a branch** for your changes
4. **Make changes** and add tests
5. **Run tests**: `npm test`
6. **Commit changes** (pre-commit hooks will run automatically)
7. **Push** and create a pull request
8. **Sign the CLA** when prompted
9. **Address feedback** from reviewers

For detailed instructions, see the [Contributing Guidelines](https://github.com/mochajs/mocha/blob/main/.github/CONTRIBUTING.md).
